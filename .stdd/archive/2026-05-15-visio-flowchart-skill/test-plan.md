## 一、测试策略

### 1.1 测试金字塔

本项目是 Claude Code Skill（SKILL.md 指令集），不是传统代码。测试以**人工行为验证**为主，辅以**结构校验脚本**：

| 层次 | 占比 | 说明 |
|------|------|------|
| 结构校验 | 15% | YAML frontmatter 合法性、Markdown 结构完整性 |
| 行为验证 | 70% | 给定输入 prompt，验证 Claude+Skill 的行为输出是否符合 spec |
| 端到端验证 | 15% | 完整链路：自然语言描述 → .vsdx 文件在 Visio 中打开检查 |

### 1.2 测试原则

- 每个 spec Scenario 至少对应 1 个 TC 案例
- 行为验证以 checklist 形式执行，每次 Skill 修改后逐项核对
- 端到端验证以生成的 .vsdx 文件在 Visio 中打开为最终判断标准
- 结构校验可自动化（Bash 脚本），行为验证需人工执行

### 1.3 已有测试资产

| 测试文件 | 用例数 | 类型 | 覆盖范围 |
|----------|--------|------|----------|
| 无 | 0 | - | 全新项目 |
