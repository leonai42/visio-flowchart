# Visio 流程图 Skill - 技术设计

## Context

- **Skill 系统**：Claude Code 的 Skill 机制，通过 SKILL.md + YAML frontmatter 的 `description` 字段实现关键词自动匹配加载
- **底层 MCP**：visio-mcp v0.1.2，基于 Visio COM 自动化，提供 22 种图表类型的模板驱动创建
- **运行环境**：仅 Windows + Microsoft Visio Desktop + Python 3.14+
- **产出物**：SKILL.md（指令集）+ standards.md（公司规范配置），纯 Markdown，无需编译

## Decisions

### 1. Skill 触发机制

**方案**：YAML frontmatter `description` 字段内嵌中文/English 触发关键词

**为什么**：Claude Code 原生支持 description 字段的语义匹配，无需额外 Hook 配置，零维护成本。关键词覆盖"流程图/泳道图/flowchart/swimlane/画流程/流程梳理"等常见说法。

**备选方案及排除原因**：
- Hook 触发（settings.json event-driven）：过度设计，Skill 系统已原生支持
- 手动 `/visio-flowchart` 调用：不符合用户"自然语言触发"的需求

### 2. 规范配置格式

**方案**：独立 `standards.md` 文件，表格驱动，与 SKILL.md 分离

**为什么**：公司文档规范随时间可能调整，分离后只需编辑 standards.md，不动 SKILL.md 指令逻辑。Markdown 表格对非开发者友好，修改颜色/字体无需理解 Skill 内部逻辑。

**备选方案及排除原因**：
- JSON/YAML 配置：语法门槛高，普通用户容易写错格式
- 硬编码在 SKILL.md 中：无法适配不同公司的规范差异

### 3. 复杂度阈值

**方案**：≤5 节点直接生成，>5 节点先列出清单给用户确认再生成

**为什么**：简单流程（如"登录：输入账号→验证→进入首页"）3-4 个节点，确认步骤纯属打扰。复杂流程（审批流 10+ 节点）需要用户确认节点是否正确，避免全图返工。5 是经验阈值，覆盖大部分简单 CRUD/登录类场景。

**备选方案及排除原因**：
- 固定一步到位：复杂流程出错代价大
- 每次都确认：简单流程增加不必要的交互摩擦

### 4. 形状规范映射

**方案**：双层查找机制 — 先查 standards.md 的覆盖项，未命中回退到 ISO 5807 默认值

**为什么**：既保证开箱即用（有 ISO 5807 默认值兜底），又允许公司规范按需覆盖。用户只需在 standards.md 中声明与 ISO 不兼容的部分，其余自动继承。

**备选方案及排除原因**：
- 纯 ISO 5807 硬编码：无法满足公司内部规范要求
- 完全由用户配置所有形状：配置负担重，大部分场景跟 ISO 5807 一致

### 5. 错误处理

**方案**：Skill 指令开头嵌入前置检查步骤，引导 Claude 先验证 visio-mcp 可用性、Visio 运行状态，失败时给出中文错误提示

**为什么**：visio-mcp 连接失败时原生报错是英文 traceback，普通用户无法理解。Skill 层提前拦截，转化为可操作的中文提示。

**备选方案及排除原因**：
- 依赖 MCP 自身报错：信息不友好，用户困惑
- 在 SKILL.md 之外写检查脚本：增加维护面，Skill 指令即可完成

### 6. 泳道图角色识别

**方案**：从用户自然语言描述中提取角色关键词（"用户/系统/管理员/财务/经理"等），按角色将节点分组到不同泳道

**为什么**：用户说"用户提交申请→经理审批→财务打款"，自然已经表达了泳道边界，Skill 自动提取角色即可，无需用户显式标注泳道归属。

**备选方案及排除原因**：
- 要求用户按"角色A：步骤1,2；角色B：步骤3"格式显式标注：增加学习成本

       │
       ▼

## Risks / Trade-offs

| 风险 | 缓解措施 |
|------|----------|
| visio-mcp 版本升级导致工具名/参数变化 | standards.md 中声明依赖版本号；Skill 指令中加入 "优先使用 get_diagram_standard 获取最新标准" |
| 自然语言解析歧义（如"审核"是处理还是判断节点） | Skill 指令规范：含"是否/如果/判断/审批通过？"关键词 → 判断节点，否则 → 处理节点 |
| 泳道图角色提取遗漏 | Skill 指令中加入角色提取的启发式规则，并在确认清单中展示角色分配，用户可纠正 |
| 复杂流程布局混乱 | 批次画形状时按从上到下、从左到右排列，节点间距 1.5 英寸，泳道间距 2 英寸 |
| standards.md 配置错误导致形状异常 | Skill 指令加入配置校验：读取后检查颜色值格式（#RRGGBB）、字体名是否有效 |
